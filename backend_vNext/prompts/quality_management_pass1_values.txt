## EXTRACTION RULES
---

### For Monitoring
1. Look in Section 5.2 (Monitoring), Section 6 (Quality Assurance)
2. Derive KRIs and QTLs if not explicit (see minimum requirements)
3. Use enum values exactly as specified

### For RBQM
1. Extract 5-7 CTQ factors (3-4 explicit + 2-3 inferred)
2. Extract 3-5 emergent systemic risks (highest impact only)
3. Extract 3-4 strategic KRIs (leading indicators ONLY)
4. Map risks to TransCelerate 13 categories
5. Use 5x5 scoring for risk matrix

### Minimum Required KRIs (always generate at least 5)
1. KRI for Informed Consent (category: protocol_compliance)
2. KRI for Primary Endpoint Data (category: data_integrity)
3. KRI for SAE Reporting (category: patient_safety)
4. KRI for Protocol Deviations (category: protocol_compliance)
5. KRI for Visit Compliance (category: protocol_compliance)

### Minimum Required QTLs (always generate at least 3)
1. QTL for Informed Consent Deviations (category: informed_consent)
2. QTL for Major Protocol Deviations (category: protocol_compliance)
3. QTL for Data Quality (category: data_integrity)

### DO NOT EXTRACT
- ANY provenance fields (section_number, page_number, text_snippet)

### OUTPUT
Return ONLY valid JSON. No markdown fences, no explanations.
