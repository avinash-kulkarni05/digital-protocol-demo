## Instructions

For EACH activity, analyze the name semantically and determine:

1. **category**: Human-readable category name (e.g., "LABORATORY", "VITAL_SIGNS", "ECG")
2. **cdashDomain**: 2-letter CDISC domain code (e.g., "LB", "VS", "EG")
3. **cdiscCode**: NCI code if known (format: Cxxxxx), or null if uncertain
4. **cdiscDecode**: CDISC preferred term, or null if uncertain
5. **confidence**: Your confidence level (0.0-1.0)
6. **rationale**: Brief explanation of why this mapping is correct

## Guidance for Edge Cases

- **Compound activities** (e.g., "Vital Signs and Oxygen Saturation"): Use the primary component (VS)
- **Alternative procedures** (e.g., "ECHO or MUGA"): Use the common domain (EG for cardiac)
- **PK/PD samples**: Use PC if pharmacokinetic focus, BS if general biomarker
- **Lab panels** (e.g., "Hematology", "Chemistry Panel"): Use LB
- **Imaging for tumor**: Use MI (imaging), not TU (tumor domain is for tumor identification data)
- **Study drug administration**: Use EX (Exposure)
- **Pregnancy tests**: Use LB (Laboratory)
- **RECIST/tumor assessments**: Use MI if imaging-based, TU if clinical assessment
