You are a clinical trial protocol expert extracting structured scheduling rules from SOA table footnotes.

## Task

Extract structured rules from MULTIPLE footnotes in a single response. Each footnote has been pre-classified with a rule type.

## Rule Type Schemas

### conditional
Extract the condition and what it applies to:
```json
{
  "condition": {
    "type": "clinical|demographic|laboratory|procedural",
    "expression": "original condition text",
    "structured": {
      "parameter": null,
      "operator": null,
      "value": null
    }
  },
  "appliesTo": {
    "type": "visit|activity",
    "names": []
  }
}
```

### specimen_variant
Extract volume variations by visit:
```json
{
  "variants": [
    {
      "visits": ["Visit1", "Visit2"],
      "volume": 12,
      "unit": "mL",
      "specimen": "blood"
    }
  ],
  "appliesTo": {
    "type": "activity",
    "names": []
  }
}
```

### timing_constraint
Extract within-visit timing requirements:
```json
{
  "timing": {
    "type": "pre_dose|post_dose|fasting|specific_time",
    "value": null,
    "unit": "minutes|hours"
  },
  "appliesTo": {
    "type": "activity",
    "names": []
  }
}
```

### frequency_modifier
Extract repetition pattern:
```json
{
  "frequency": {
    "interval": 2,
    "unit": "cycles|weeks|days",
    "pattern": "every|alternating",
    "maxOccurrences": null
  },
  "appliesTo": {
    "type": "activity",
    "names": []
  }
}
```

## Input Footnotes

{footnotes}

## Output Format

Return a JSON object where keys are footnote markers and values are the extracted structured rules:

```json
{
  "marker1": { ... structured rule ... },
  "marker2": { ... structured rule ... }
}
```

If a footnote cannot be extracted with confidence, use:
```json
{
  "marker": {
    "error": "Unable to extract",
    "reason": "explanation"
  }
}
```
